Эту сборку я развернул на сервере в домашней сети.
Склонировал репозиторий, установил nodejs, установил docker engine с плагином docker compose, установил зависимости yarn install.
Скопировал пример .env 
```sh
cp .env.template .env  
``` 
в нем закомментировал 
```sh
# REDIS_URL=
# DATABASE_URL=
# DB_NAME=
```
т.к. эти переменные определены в docker-compose.yml
и запустил контейнер yarn docker:up.
При вводе 
docker compose logs -f
получаю ответ 
✔ Server is ready on port: 9000 - 3ms
info:    Admin URL → http://localhost:9000/app
Т.к. ip сервера в домашней сети http://192.168.31.85:9000 открываю панель администратора http://192.168.31.85:9000/app
Впервые ввожу почту и пароль и получаю ответ 
Welcome to Medusa 
Sign in to access the account area
Что не так? Помоги разобраться


home-srv% docker compose ps
NAME      IMAGE     COMMAND   SERVICE   CREATED   STATUS    PORTS
home-srv% docker compose rm -f
No stopped containers

1. Проверьте запущенные контейнеры:
    cd /home/sacred/medusa-init/medusa-store && docker compose ps
2. Если есть запущенные контейнеры, остановите их:
   cd /home/sacred/medusa-init/medusa-store && docker compose down
3. Удалите все образы, связанные с этим проектом:
   cd /home/sacred/medusa-init/medusa-store && docker image prune -a -f
4. Выполните очистку системы Docker:
   cd /home/sacred/medusa-init/medusa-store && docker system prune -f
5. Выполните очистку данные базы данных (это удалит все данные, включая созданного пользователя):
   cd /home/sacred/medusa-init/medusa-store && docker volume prune -f
6. Затем заново запустите контейнеры:
   cd /home/sacred/medusa-init/medusa-store && yarn docker:up
